## 1. 要设计的屏（全景）

```
今日页 (tab0, 新)  ──「日程/非日程」toggle──┐
   ├ 晨报 hero(上午首开,折叠)              │
   ├ header(日期 · ⚡N闪念 · 📅 · toggle)   │
   └ 非日程=段视图(默认)  ⇄  日程=24h网格    │
                                            │
日历(📅 进入) ── 流(逐日 compact 段视图) ──┤── 点某天 → DayDetail(默认进"日程"网格)
                月 / 年                     │
                                            │
当日闪念视图(DayFlashView) ←── 点 header 的 ⚡N闪念 pill（任意屏）
```

**核心概念:一天 = 两个模式,一个 toggle(命名固定为「日程 / 非日程」)**
- **非日程** = 上午/下午/晚上 **分段视图**（捕捉向，"我今天记/发生了啥"）。
- **日程** = **24h 网格**（日程向，"把这天当日程表看"；事件画成时长块）。

## 2. 逐屏设计 brief

### 2.1 段视图（「非日程」）—— 本次最核心的新视觉

一天的内容，按 **5 个时段**分组（空段不画）：

| 段 | 时间 | 段头建议图标 |
|---|---|---|
| 凌晨 | 00:00–05:59 | 🌙 |
| 上午 | 06:00–11:59 | 🌅 |
| 中午 | 12:00–12:59 | ☀️ |
| 下午 | 13:00–17:59 | 🌆 |
| 晚上 | 18:00–23:59 | 🌃 |

每段结构：**轻段头** → 有钟点的卡（按时刻升序、卡上显 `HH:MM`） → **「没具体时间」组**（用户只说了"早上"没说点的，轻量收在段尾、不显时间）。
- **顶部「全天」条**：全天事件（放假 / 团建；多数日子没有）。
- **底部「没说时间」组**：归到这天、但没说钟点也没时段的记录（典型：今天说"昨天买衣服120 / 前天打球180"→落到昨天 / 前天的这个组）。**措辞要软，别叫"待安排"**（过去的消费不是待办）。
- 卡 = 复用 SkillCard DNA + 领域 chip。
- **空段折叠**：轻日子可能只有一两段 —— 要让"只有下午两条"也好看、不空洞。

**必须打中**：
- 读起来是**一条会呼吸的时间线**，不是分组表格。段头要轻（像晨昏标记，不像 section 标题）。
- "没具体时间"组要**柔**：别用 "无具体时间" 这种工程词当大标题；用很淡的分隔 / 留白即可（措辞你定，建议像"还有 · 没说时间"或纯视觉分隔）。
- 轻日子（1 段）和重日子（5 段全满）都成立。

**雷区**：别做成 5 个生硬的卡片分区；别让空段留占位。

### 2.2 今日页（tab0，新首页）

自上而下：
1. **晨报 hero**（上午首开、可折叠）：早安问候 + 今日预览 + 昨日回顾 + 本周进度。**复用已有设计包 `morning-brief-a/b.html`**（A 日出暖橙 / B 黎明冷蓝，按日交替；hero 挂真实 Reka）—— 这块已有皮，**今日页要把它收成顶部 hero**（✕ /「开始今天」收起，当天不再展开），而不是独立整屏。
2. **header 行**：日期(今天) · **`⚡ N 条闪念` pill** · **📅**(去日历) · **「日程/非日程」toggle**。
3. **正文**：默认 **非日程 段视图**（§2.1）渲染今天；toggle 可切今天的「日程」网格。
- **空状态**（新用户 / 今天还没记）：温柔引导（"今天还没记什么，说一句?"）+ Reka 在场感。**这是新用户第一印象，重点设计。**

**必须打中**：今日页要"**值得每天早上打开**"；hero 暖、正文是今天的真实生活流。
**雷区**：今日页**不是 Reka 仪表盘**（Reka 的主动提示留在浮球 + 我的岛，不堆这里）；默认**不是**网格。

### 2.3 「日程/非日程」toggle

- 两态切换控件，放 header。命名固定 **「日程」/「非日程」**。
- 设计它的视觉 + 两态切换的微动效（段视图 ⇄ 网格）。要小、清楚、不抢戏。

### 2.4 闪念入口 pill + 当日闪念视图（DayFlashView）

- **入口 = 一颗 `⚡ N 条闪念` pill，挂在「那天的 header」**（今日页 header / 流的每日 tile 头 / DayDetail 顶部）。**「日程」「非日程」两模式位置一致**、不沉底、不随模式跑。N=0 不显。
- 点它 → **当日闪念视图**：把那天的闪念捕捉聚到一处，**时间倒序**列每条（时刻 + transcript 摘要 + 它产出的卡作 provenance），点一条 → 进该捕捉的回放 session。
- **背景**：闪念原来混在流里显得乱，所以**移出流、收成这颗 pill**。设计要让它"安静但找得到"。

**必须打中**：pill 克制（不是大按钮）；DayFlashView 像"今天随口说过的一摞便签"，轻、快、可回看。

### 2.5 流（ScheduleView，经 📅 进入）

- 一条竖向滚动的"逐日"流；**每天一个 tile = compact 段视图**（段头 + 内容，紧凑；太多可"+N 更多"）。
- 每个 tile **头部** = 日期 + `⚡N` 小徽标。
- **空日** = 点选式快记（点中→露出"+ 在这天记一笔"，不常驻按钮）。
- 点某天空白/日期 → 进 **DayDetail**。

**必须打中**：能快速**回看很多天**（紧凑），又一眼看出每天的"形状"（哪段有事）。

### 2.6 DayDetail（从流点进某天）

- **默认进「日程」= 24h 网格**（因为钻进某天 = 想看精确日程）；toggle 可切回非日程。
- **24h 网格已实现**（24 行 × 56px 时刻轴），本次**保留** —— 你可做**视觉精修**：事件按 `start–end` 画**时长块**（3–5 点的会 = 2h 块）、全天事件顶部条、无时刻捕捉走顶部类型 tab 区。

**这正是和"流/今日页(非日程)"的差异**：非日程里 3 点的会只是一条记录；日程网格里它是一个 2 小时的块。设计上要让这两个"镜头"各自成立、切换顺滑。

### 2.7 晨报 hero（承 §2.2 第 1 点）

已有 `morning-brief-a/b` 两套沉浸皮 —— 本次任务 = **把它从"独立整屏"改造成"今日页顶部 hero 段"**：收起态、展开态、与下方段视图的衔接。

## 4. 你来定的设计决策（design's call）

1. 段头的视觉重量 + 是否显时间区间（建议只显"下午"，不显"13:00–18:00"）。
2. "没具体时间"组的呈现（淡分隔 / 缩进 / 措辞），**去工程味**。
3. `⚡N闪念` pill 的形态与位置微调（header 左/右）。
4. 「日程/非日程」toggle 的控件样式 + 切换动效。
5. 今日页**空状态** + Reka 在场的表达。
6. 今日页是否给"当前时段"一点"现在"高亮（可选，加分项）。
7. 流 compact tile 的密度（显几条 / 怎么"+N"）。
8. 晨报 hero 收起↔展开的过渡。
9. 浅色（暖白）主题稿是否同出。

## §B · DayDetail「日程」= 记录类型容器 + 24h 网格放待办（⏳ 待实现 · Part B）

> 源:用户 `流与待办-Handoff.dc.html` Part B + 线框 `日历改版线框.dc.html` ③。**这是流/月落地后的下一步。** 规则真值同 §4.5.4。

**B1 进入 / 镜头**:从流点日期 → 默认落「**日程**」镜头(想看精确时间安排);顶部「非日程 / 日程」两态 toggle(280ms 交叉淡入);⚡闪念 pill 位置与流一致(顶部)。

**B2 什么进网格**:**只有「日程(事件)+ 待办」进 24h 网格。** 结果记录(网球 / 记账 / 睡眠 / 带娃…)**不**进网格,收进顶部「**记录 · 按类型**」**定高滚动容器**(按类型 tab 分类,超出内部滚动,卡走 3 行 DNA)。网格顶部两类**轻条**(不占时间轴):**全天**事件、**待安排**(无时刻待办)。一条「**现在**」红线(`#F7768E`)横切当前时刻。

**B3 待办的 4 种展示**:
1. **有时刻的单个待办** → 网格内小块(○ 勾选框 + 标题),落它的时刻行。
2. **点状待办撞长事件** → 待办缩成**带标题的瘦长 chip**(○ 办理值机)占重叠时段左半、事件占右半(不抢列宽,一眼知是什么待办、非裸圆点)。
3. **同一时刻 N 个待办** → 收成「**N 个待办 ▾**」计数 chip,点开就地展成小列表(逐条带勾选框),不占多列、不遮事件。
4. **无时刻待办(待安排)** → 收到网格顶部「**待安排**」轻条,**最多 3 条** + 头部「共 N 条」+「展开其余 N 条 ⌄」(展开后内部滚动、不无限撑高)。

**B4 重叠规则**:
- 待办点(14:00)+ 事件(14–20)→ 待办 = 左半瘦 chip(带标题)、事件 = 右半整列。
- 事件(14–20)+ 事件(14–15)→ 重叠时段**等分两列**;15:00 后短事件结束,长事件**不自动变宽**(保列位避免跳动)、右列空出。
- N 个并发 → 重叠区**均分 N 列**;超过 3 列收成「+N」竖排省略、点开看全部。
- **镜头一致性**:同一条飞深圳,「非日程」= 一条记录、「日程」= 16–18 的 2h 时长块,两镜头切换顺滑对应。

**B5 数据(无需新表,现有字段即可)**:`Event` = `kind=='event'`(start/end/all_day);`Todo` = skill `todo` 的 asset(`occurred_at`→time、`payload.status`→done);`Record` = 其余 asset。网格只渲 Event + Todo,Record 进顶部类型容器。重叠 = 按 `[start,end)` 求并发分组 → 组内均分列宽。`domain` 已在 timeline item(§3.8)。
