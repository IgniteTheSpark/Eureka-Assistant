# Design: UReka 习惯 / streak 系统 + 三层核心价值

> office-hours(startup mode)产出 · 2026-06-25 · Status: **DRAFT**
> 这是**产品/价值设计稿**,不是实现卡。落点指向 §4.5.0 / §14 / §7 / §9 / §2 / §4.8(见末尾「接入点」)。

## 1. 问题
留存。app 现在**没有稳定的「明天还回来」的理由**。捕捉很爽(wedge),但**记录完没有正反馈、不形成习惯**(用户原话)。现有闭环「攒够量 → insight/报告」太慢太理性 —— 它是「**深化**」循环(让已留下的人更黏),不是「**拉回**」循环。

## 2. 证据 / 现状
- 用户自评:**留存本就低 / 说不清**(office-hours D1)。
- 用户原话(pitch):**「记录完了没有正反馈,记录没有习惯。」**
- 现有「深」闭环:§14 积累 offer → 报告(已实现,但拉回力弱)。
- 竞品格局:纯习惯 app 有紧日循环但**输入摩擦大**;纯日记 app 输入轻但**无日拉回**。UReka **同时有捕捉层 + 宠物** = 两边都结构性做不了的组合。

## 3. 核心价值(三层)
1. **钩子(wedge)= 零摩擦输入的爽** → 获客 + 第一次惊艳。
2. **日循环(留存,← NEW,本稿重点)= capture-fed 习惯 / streak** → 每天那个「它真的在帮我变好」的看得见证据。
3. **深循环 = insight / 报告** → 让老用户更黏。

> **Reka 即时反应 + 今日页仪式感 = 横切质量层**(交互精进 + UIUX 升级),贯穿三层、不是独立支柱(office-hours D2 用户修正)。

一句话价值:**「你只管说,它替你把生活记好、并悄悄帮你变好。」**

## 4. 模型(已定)
- **三类内容**:**random 一次性**(事件/名片/待办/外部/随记)· **模版**(= **一个自定义技能本身**,如「宝宝打针记录」,无新机制)· **习惯**(= 模版 **+ 频率/逻辑/目标 config + streak**)。
  > 关键:**「模版」不是新东西 = 自定义技能现有能力**;**唯一新增 = 习惯那一层 config + streak 状态**。新增面很小。
- **习惯 = 可挂在任何技能上的可选层**(模型通用)。**MVP 仅对自定义技能开放「设为习惯」,常驻/baseline 技能暂屏蔽**(D3,后续解锁「每天记账」不改模型)。
- **创建后再定义**:建技能时只是个记录类型 → 之后在详情/设置「**设为习惯**」才挂上 频率/逻辑/目标。**Reka 可从 §14 节律主动促成**(「你每周都记网球,设成习惯?」)。
- **频率**:**每周 N 次(弹性)** + **固定周几(刚性)**;每天 = 每周 7 次特例。
- **逻辑(算「完成一次」)**:MVP = **capture-fed**(有一条该技能记录 = 打卡 ✓,零摩擦、承 wedge);**阈值**(喝水习惯需「≥8 杯」)后置。
- **目标 — 两型**:
  - **开放(cadence)**:无限期保持节奏,只有 streak,没有「完成」。
  - **有限(challenge)**:N 天 / 累计 N 次 / 到某日 → **完成 = Reka 庆祝 + 收官**,然后「**再来一轮 / 加难度 / 存档**」(完成可再开启)。
- **streak**:普通连续 streak;**断了 = 「可惜」非「愧疚」**(用户定:不做归零保护那套过度设计);Reka 保持善意;**里程碑给奖励 → 接 §9 装饰/掉落经济**。
- **打卡 → 喂 §7/§9 `completion_event` → 周岛/里程碑**。**习惯 = 日级即时层,周岛 = 周级深化层** —— 刚好补上「捕捉那一秒」缺的即时反馈。

## 5. 三个小细节(先按推荐默认,待确认)
1. **一天多次打卡**:cadence 习惯**一天一次封顶**(同日多记不重复计);**累计型 goal 每条计**。
2. **今日页并存**:**有时间习惯 → 今日安排卡**、**无时间习惯 → Reka Offer 卡**,卡上带 **streak + 目标进度**。**无新面**(§4.5.0 已有位置)。
3. **数据落点**:新建**轻量 habit 实体**(`skill_ref + frequency + logic + goal + streak_state`),completion **复用现有 `completion_event` 管线**(不另起炉灶)。

## 6. 别做
- 别做**手动每日打卡器**(自己每天点打卡 = 背叛 wedge)。
- 别做**归零愧疚机制**(用户只「可惜」,普通 streak 即可)。
- 习惯**别自动从 random 捕捉长出**(默认关;Reka 只对真有节律的主动建议)。

## 7. 开放问题
- baseline 习惯何时解锁(「每天记账/每天随记」)。
- 阈值逻辑(如喝水≥8杯)的配置 UI。
- 习惯卡与今日页「有时间的习惯」的视觉合并细节(交 design)。

## 8. 成功标准
- **主指标:D1 / D7 留存随 capture-fed 习惯上线而上移。**
- 设过 ≥1 个习惯的用户占比 + 「设了习惯」用户的次日回访率(对照没设的)。

## 9. Assignment(下一步,不是「去 build」)
1. **先拉现有真实 D1/D7 留存数字**,坐实「留存本就低」(D1 现在是体感)。
2. 从当前用户**最高频的捕捉类型**里挑 **1 个**(很可能是记账或随记),做 capture-fed streak 的**最小试点**,看留存是否动。**证据先行,再全量。**
