# 一、产品概述

## 1.1 产品背景

UReka Calendar 不是只展示事件的传统日历。它将 Event、Todo、随记、名片和闪念来源统一放进同一条时间轴，帮助用户同时回答三个问题：

1. 最近每天发生或记录了什么；
2. 某一天有哪些需要按时间执行的安排；
3. 某条内容从哪里产生，并能否继续讨论、编辑、完成或删除。

Calendar 必须同时支持“回看记录”和“查看日程”两种心智。前者使用时段分组，强调一天的内容形状；后者使用 24 小时网格，强调事件时长、待办时间和冲突关系。

## 1.2 产品定位

Calendar 是 Todo、随记、Event、名片和闪念的统一时间视图。

- 它不要求所有内容都具备开始和结束时间。
- 它不把闪念原始输入混入普通内容列表。
- 它不为四种内容分别设计不同的弹层框架，而是复用统一的详情 Bottom Sheet。
- 它不在 Calendar 顶层提供常驻“新增事件”按钮；指定日期创建放在空日引导和每日详情页中。

## 1.3 产品目标

1. 用户进入 Calendar 后直接看到今天附近的内容，而不是先选择日期。
2. 用户可在流、月、年三个观察尺度间切换，并保持明确的下钻关系。
3. 同一天的内容按“非日程”和“日程”两种方式查看。
4. 点击任意内容后，进入一致的详情与操作体验。
5. 内容修改后所有 Calendar 表面自动刷新，且刷新期间不闪空。
6. 另一团队可依据本文独立复刻交互、分桶规则和异常状态，并自行决定技术实现。

# 二、典型场景

1. 用户打开 Calendar，默认落在“流”，并自动定位到今天。
2. 用户向前或向后滚动，快速回看连续多天的内容。
3. 用户在月视图选择某日，通过底部内容区预览当天记录，再进入每日详情。
4. 用户从年视图选择一个月，切回月视图继续下钻。
5. 用户进入每日详情，用“非日程”看分时段记录，用“日程”看全天事件、待安排待办和 24 小时网格。
6. 用户点击 Event、Todo、随记或名片，在统一详情 Bottom Sheet 中继续讨论、编辑或删除。
7. 用户直接勾选待办完成，并立即看到 Calendar 中的完成状态变化。
8. 用户点击 `⚡N`，回到当天闪念会话查看原始捕捉上下文。

# 三、信息架构与页面流转

```text
Calendar
├── 流（默认，自动定位今天）
│   ├── 点击有内容日的日期/空白 → 每日详情
│   ├── 点击空日 → 展开创建引导 → 指定日期创建
│   ├── 点击内容条目 → 详情 Bottom Sheet
│   └── 点击 ⚡N → 当天闪念会话
├── 月
│   ├── 首次点击日期 → 选中日期并刷新底部预览
│   ├── 再次点击已选日期 → 每日详情
│   ├── 点击底部日头/空白 → 每日详情
│   ├── 点击内容条目 → 详情 Bottom Sheet
│   └── 点击 ⚡N → 当天闪念会话
└── 年
    └── 点击月份 → 切换到该月的月视图

每日详情
├── 非日程 → 分时段内容
├── 日程 → 记录区 + 全天/待安排 + 24h 网格
├── 点击条目 → 详情 Bottom Sheet
├── 待办 ○ → 完成/撤销完成
└── 在这天记一笔 → 指定日期创建菜单

详情 Bottom Sheet
├── 讨论 → 带当前内容上下文的 Chat
├── 编辑 → 对应类型的全屏编辑页
├── 完成/撤销完成 → 仅可勾选待办
└── 删除 → 二次确认 → 删除并刷新
```

# 五、全局基础规则

## 5.1 Calendar 内容类型

| 类型 | Calendar 中的展示 | 详情与操作 |
|---|---|---|
| Event | 按开始时间进入日期和时段；全天 Event 进入全天区域 | 专用 Event 详情，可讨论、编辑、删除 |
| Todo | 有钟点时进入时间位置，无钟点时进入“待安排” | 可完成/撤销完成、讨论、编辑、删除 |
| 随记 | 按记录所属日期进入时段或“没有时间” | 可讨论、编辑、删除 |
| 名片 | 按记录所属日期进入时段或“没有时间” | 可讨论、编辑、删除 |
| 闪念 | 不作为普通条目出现，只以当天 `⚡N` 入口展示 | 点击直接进入当天对应的 Session Page |

自定义技能、记账及其他类型不在本次复刻范围内，不需要设计相关展示或详情。

## 5.2 日期落点与时段分桶

### 5.2.1 自然日落点

Calendar 按用户本地日期将内容归到对应自然日。

| 内容 | 日期落点规则 |
|---|---|
| Event | 按开始日期落点 |
| Todo | 优先按用户指定的发生日期或截止日期；未指定时按记录日期 |
| 随记 | 按记录日期；从某日创建时归到该日 |
| 名片 | 按记录日期；从某日创建时归到该日 |
| 闪念 | 按捕捉发生日期统计到当天 `⚡N` |

### 5.2.2 时段定义

| 时段 | 时间范围 |
|---|---|
| 凌晨 | 00:00-05:59 |
| 上午 | 06:00-11:59 |
| 中午 | 12:00-12:59 |
| 下午 | 13:00-17:59 |
| 晚上 | 18:00-23:59 |
| 没有时间 | 用户没有表达钟点或时段，只能确定所属日期 |

### 5.2.3 分桶优先级

1. 原始闪念先排除，归入 `⚡N`。
2. 用户只表达凌晨、上午、中午、下午或晚上时，按该时段分组。
3. 用户表达明确钟点时，按钟点对应的时段分组。
4. 其余进入“没有时间”。
5. 只有时段、没有钟点的条目位于对应时段尾部，但不显示伪造时间。
6. Event 在非日程模式只使用开始时间，在日程模式使用开始与结束表达时长。

## 5.3 日程模式分桶

| 内容条件 | 去向 |
|---|---|
| 全天 Event | 全天托盘 |
| 非全天 Event | 24 小时网格 |
| 有明确钟点的 Todo | 24 小时网格 |
| 无明确钟点的 Todo | 待安排托盘 |
| 闪念 | 页头 `⚡N`，不进入正文 |
| 随记、名片 | 记录区，按类型筛选 |

## 5.4 空值与边界规则

1. 标题为空时，各表面按类型回退为“事件”“待办”“记录”或“闪念”。
2. `⚡N` 的 N 为当天闪念数量，0 时不展示。
3. 非本月日期可选，选择后底部预览对应真实日期。
4. 已完成 Todo 不从 Calendar 隐藏。
5. 删除失败时不得在客户端永久移除条目。

# 六、内容刷新、加载与异常

## 6.1 内容刷新规则

1. Calendar、每日详情和详情 Sheet 中展示的是同一份用户内容。
2. 创建、编辑、删除或修改 Todo 完成状态后，用户返回 Calendar 时必须看到最新结果。
3. 刷新期间继续展示上一次成功内容；新内容加载完成后再替换，避免页面闪空。
4. 详情 Sheet 应展示完整内容，不应只保留 Calendar 列表中的简短摘要。
5. 具体同步机制和数据组织方式由复刻团队决定。

## 6.2 异常处理

| 异常 | 当前行为 | 复刻要求 |
|---|---|---|
| Calendar 首次加载失败 | 显示错误文本 | 必须提供明确错误态 |
| 已有内容后的刷新失败 | 保留当前已展示内容 | 不得闪空或整页退出 |
| 条目详情加载失败 | 有来源会话时回退来源会话 | 无来源会话时保持当前页面，不崩溃 |
| Sheet 完整内容加载失败 | 保留当前可见摘要 | 不关闭 Sheet |
| Todo 完成状态保存失败 | 恢复原状态 | 不得让用户误以为已经保存成功 |
| 删除失败 | 条目保持原位或保留 Sheet | 不得让条目假消失 |
| 事件保存失败 | 表单内显示错误并恢复可操作状态 | 不丢输入内容 |
| 闪念没有可打开的 Session | 点击无跳转 | 不打开空页面 |

# 七、验收标准

## 7.1 主路径验收

1. 新安装打开 Calendar，默认进入流并定位今天。
2. 点击月、年或左右滑动，可切换视图，切换器状态一致。
3. 在远端日期再次选择流，回到今天。
4. 流中点击有内容日空白进入每日详情；点击条目只开对应详情。
5. 空日第一次点击只出现引导，第二次点击才进入指定日期创建。
6. 月视图第一次点击日期只选中，第二次点击同日进入每日详情。
7. 年视图点击月份后进入该月的月视图。
8. 每日详情默认日程；切换非日程后展示分时段记录。
9. 日程模式正确区分记录、全天、待安排和网格内容。
10. 点击 Event、Todo、随记或名片后，Sheet 可在 62% 与 95% 高度间拖动吸附。

## 7.2 时间与分桶验收

1. “下午开会”没有钟点时，不显示伪造的 15:00。
2. “下午 3 点开会”显示 15:00，并进入下午时段。
3. 全天事件只进入全天托盘，不进入小时网格。
4. 无时刻待办进入待安排，有时刻待办进入网格。
5. 随记和名片只进入记录区，不进入网格。
6. 同时刻多个待办折叠成计数块；展开后下方时间内容整体下移。
7. 重叠事件等宽分列，无相互遮挡。
8. 已完成待办仍展示，且有绿勾与删除线。
9. 今天显示当前时间线，其他日期不显示。
10. 用户选择的日期时间在保存和重新打开后保持一致，不发生跨天偏移。

## 7.3 详情与写操作验收

1. Event、Todo、随记和名片分别打开对应详情。
2. 闪念直接进入会话，不打开 Bottom Sheet。
3. Todo 可从网格、待安排和详情 Sheet 切换完成状态。
4. 完成状态保存失败时视觉状态回滚。
5. 编辑成功后返回 Calendar，数据自动刷新且页面不闪空。
6. 删除必须二次确认；失败时条目保留。
7. 讨论打开带当前内容上下文的 Chat。
8. Event 编辑使用专用时间表单，非全天结束时间必须晚于开始时间。
9. Sheet 完整内容加载失败时仍能展示 Calendar 中已有的摘要。
