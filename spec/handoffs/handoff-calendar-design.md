## 2. 逐屏设计 brief

### 2.1 段视图（「非日程」）—— 本次最核心的新视觉

一天的内容，按 **5 个时段**分组（空段不画）：

| 段 | 时间 | 段头建议图标 |
|---|---|---|
| 凌晨 | 00:00–05:59 | 🌙 |
| 上午 | 06:00–11:59 | 🌅 |
| 中午 | 12:00–12:59 | ☀️ |
| 下午 | 13:00–17:59 | 🌆 |
| 晚上 | 18:00–23:59 | 🌃 |

每段结构：**轻段头** → 有钟点的卡（按时刻升序、卡上显 `HH:MM`） → **「没具体时间」组**（用户只说了"早上"没说点的，轻量收在段尾、不显时间）。
- **顶部「全天」条**：全天事件（放假 / 团建；多数日子没有）。
- **底部「没说时间」组**：归到这天、但没说钟点也没时段的记录（典型：今天说"昨天买衣服120 / 前天打球180"→落到昨天 / 前天的这个组）。**措辞要软，别叫"待安排"**（过去的消费不是待办）。
- **卡 / item 见 §C** —— 流 / 月 每条 item 收成 `[时间][类型emoji] 标题① · 标题②? · 领域tag`(标题①② = render_spec 的 ≤2 标题行字段);详情卡才用完整 SkillCard DNA。
- **空段折叠**：轻日子可能只有一两段 —— 要让"只有下午两条"也好看、不空洞。

**必须打中**：
- 读起来是**一条会呼吸的时间线**，不是分组表格。段头要轻（像晨昏标记，不像 section 标题）。
- "没具体时间"组要**柔**：别用 "无具体时间" 这种工程词当大标题；用很淡的分隔 / 留白即可（措辞你定，建议像"还有 · 没说时间"或纯视觉分隔）。
- 轻日子（1 段）和重日子（5 段全满）都成立。

**雷区**：别做成 5 个生硬的卡片分区；别让空段留占位。

### 2.2 今日页（tab0，新首页）

**今日页 = 首页 landing，两块**（**hifi 真值 = 用户 prototype `design_handoff_today_page`；详见 §E**）—— **不是今天的段视图**（回看 = §2.5 / §2.6）。两块面板浮在**全屏物理气泡场**之上，Reka 浮球在最上层。

- **Part 1 · Next Action**：**扇形可滑卡叠**（事件带每秒倒计时 + 进度、待办带完成 ✓；下方无时间待办可展开）；**早报 merge 进来**。
- **Part 2 · Recorded = Dashboard + 气泡池**：气泡池 = 今天**按创建时刻**收的 asset，一条一颗（**域色填充 + 类型 emoji**，§8 八色），点开 / 拖抛 / 倾斜滚 / 新记录 drop-in；顶部可折叠 **Dashboard**（类型筛选 chips + 速览条 + 玫瑰 / 柱状 / 树图，全部按类型→选类按领域）。**闪念 = `⚡N` pill、不进池、不是记录类型**。
- **Reka 不当主角** —— 全局悬浮小宠物、不居中。

**质感铁律**：高级（材质 / 光 / 深度，不卡通）· 直观（一眼看清 接下来 + 今天记了多少）· 有交互（卡叠滑、图表切、气泡 点 / 拖 / 倾斜 / drop-in）。
**雷区**：别做成卡片列表；别让 Reka 占首页中心；**别照 demo 调色板 / 别把闪念当记录类型**（对齐见 §E）。

### 2.3 「日程/非日程」toggle

- 两态切换控件，放 header。命名固定 **「日程」/「非日程」**。
- 设计它的视觉 + 两态切换的微动效（段视图 ⇄ 网格）。要小、清楚、不抢戏。

### 2.4 闪念入口 pill + 当日闪念视图（DayFlashView）

- **入口 = 一颗 `⚡ N 条闪念` pill，挂在「那天的 header」**（今日页 header / 流的每日 tile 头 / DayDetail 顶部）。**「日程」「非日程」两模式位置一致**、不沉底、不随模式跑。N=0 不显。
- 点它 → **当日闪念视图**：把那天的闪念捕捉聚到一处，**时间倒序**列每条（时刻 + transcript 摘要 + 它产出的卡作 provenance），点一条 → 进该捕捉的回放 session。
- **背景**：闪念原来混在流里显得乱，所以**移出流、收成这颗 pill**。设计要让它"安静但找得到"。

**必须打中**：pill 克制（不是大按钮）；DayFlashView 像"今天随口说过的一摞便签"，轻、快、可回看。

### 2.5 流（ScheduleView，经 📅 进入）

- 一条竖向滚动的"逐日"流；**每天一个 tile = compact 段视图**（段头 + 内容，紧凑；太多可"+N 更多"）。
- 每个 tile **头部** = 日期 + `⚡N` 小徽标。
- **空日** = 点选式快记（点中→露出"+ 在这天记一笔"，不常驻按钮）。
- 点某天空白/日期 → 进 **DayDetail**。

**必须打中**：能快速**回看很多天**（紧凑），又一眼看出每天的"形状"（哪段有事）。

### 2.6 DayDetail（从流点进某天）

- **默认进「日程」= 24h 网格**（因为钻进某天 = 想看精确日程）；toggle 可切回非日程。
- **24h 网格已实现**（24 行 × 56px 时刻轴），本次**保留** —— 你可做**视觉精修**：事件按 `start–end` 画**时长块**（3–5 点的会 = 2h 块）、全天事件顶部条、无时刻捕捉走顶部类型 tab 区。

**这正是和"流/今日页(非日程)"的差异**：非日程里 3 点的会只是一条记录；日程网格里它是一个 2 小时的块。设计上要让这两个"镜头"各自成立、切换顺滑。

### 2.7 晨报 hero（承 §2.2 第 1 点）

已有 `morning-brief-a/b` 两套沉浸皮 —— 本次任务 = **把它从"独立整屏"改造成"今日页顶部 hero 段"**：收起态、展开态、与下方段视图的衔接。

## 4. 你来定的设计决策（design's call）

1. 段头的视觉重量 + 是否显时间区间（建议只显"下午"，不显"13:00–18:00"）。
2. "没具体时间"组的呈现（淡分隔 / 缩进 / 措辞），**去工程味**。
3. `⚡N闪念` pill 的形态与位置微调（header 左/右）。
4. 「日程/非日程」toggle 的控件样式 + 切换动效。
5. 今日页**空状态** + Reka 在场的表达。
6. 今日页是否给"当前时段"一点"现在"高亮（可选，加分项）。
7. 流 compact tile 的密度（显几条 / 怎么"+N"）。
8. 晨报 hero 收起↔展开的过渡。
9. 浅色（暖白）主题稿是否同出。

> - **流 / 月每条 item = 悬浮卡**(去边框 + 暖色软阴影浮在 day 容器上;无时刻的用更淡填充 + 无阴影,`_card`);**领域 = 右对齐小色点**(`Expanded` 标题把点推到卡片最右、各条对齐;原色块 pill,领域名仍在详情可读)。
> - **浅色主题加深度**:bg 暗一档(`F4F2EC`→`EFEBE1`,低于 surfaceRaised)+ day 容器暖色软阴影 → 修"beige-on-beige 扁平"(`eureka_colors.light`)。
> - **联系人详情 hero 头像 = 字母 monogram**(名字 hash 派生色环),替灰色 👤(`asset_detail_sheet._monogramAvatar`)。

**B1 进入 / 镜头**:从流点日期 → 默认落「**日程**」镜头(想看精确时间安排);顶部「非日程 / 日程」两态 toggle(280ms 交叉淡入);⚡闪念 pill 位置与流一致(顶部)。

**B2 什么进网格**:**只有「日程(事件)+ 待办」进 24h 网格。** 结果记录(网球 / 记账 / 睡眠 / 带娃…)**不**进网格,收进顶部「**记录 · 按类型**」**定高滚动容器**(按类型 tab 分类,超出内部滚动,卡走 3 行 DNA)。网格顶部两类**轻条**(不占时间轴):**全天**事件、**待安排**(无时刻待办)。一条「**现在**」红线(`#F7768E`)横切当前时刻。

**B3 待办的 4 种展示**:
1. **有时刻的单个待办** → 网格内小块(○ 勾选框 + 标题),落它的时刻行。
2. **点状待办撞长事件** → 待办缩成**带标题的瘦长 chip**(○ 办理值机)占重叠时段左半、事件占右半(不抢列宽,一眼知是什么待办、非裸圆点)。
3. **同一时刻 N 个待办** → 收成「**N 个待办 ▾**」计数 chip,**点开 = 手风琴**:撑开时间块、把下方内容整体下推(逐条带勾选框),**不悬浮覆盖、不遮事件**。
4. **无时刻待办(待安排)** → 收到网格顶部「**待安排**」轻条,**最多 3 条** + 头部「共 N 条」+「展开其余 N 条 ⌄」(展开后内部滚动、不无限撑高)。

**B4 重叠规则**:
- 待办点(14:00)+ 事件(14–20)→ 待办 = 左半瘦 chip(带标题)、事件 = 右半整列。
- 事件(14–20)+ 事件(14–15)→ 重叠时段**等分两列**;15:00 后短事件结束,长事件**不自动变宽**(保列位避免跳动)、右列空出。
- N 个并发 → 重叠区**均分 N 列**;超过 3 列收成「+N」竖排省略、点开看全部。
- **镜头一致性**:同一条飞深圳,「非日程」= 一条记录、「日程」= 16–18 的 2h 时长块,两镜头切换顺滑对应。

**B5 数据(无需新表,现有字段即可)**:`Event` = `kind=='event'`(start/end/all_day);`Todo` = skill `todo` 的 asset(`occurred_at`→time、`payload.status`→done);`Record` = 其余 asset。网格只渲 Event + Todo,Record 进顶部类型容器。重叠 = 按 `[start,end)` 求并发分组 → 组内均分列宽。`domain` 已在 timeline item(§3.8)。

## §C · item 展示 = 标题①② + wizard 双预览 / regenerate（2026-06 讨论定 · 新需求）

> 起因:截图里流/月的 item 是「一个模子 `[时间][emoji][文字]`」,不 intuitive —— 测试随记和真会议长一样、内容太薄(如「💰 125」)。真值:[§4.5.0a](../04-frontend.md)(流行内容)+ [§4.8](../04-frontend.md)(wizard)。

### C.1 流 / 月 每条 item = 「标题行」

`[时间] [类型 emoji] 标题① · 标题②? · [领域 tag]`（带边框、段色温）

- **标题①②** = 该技能 render_spec 的 **≤2 个标题行字段**:记账 `💰 ¥125 · 早餐`(金额·用途)、待办 `📌 带宝宝去打针`(任务 1 个)、随记 `✍️ 内容摘要`(1 个)。要一眼读懂"花了多少、买了啥",不是干巴巴 `💰 125`。
- **时间打头**(保留,用户要)、**类型靠 emoji**(选准:事件用 📅 别用 👤)、内容 = 1~2 格关键字段。
- **副标题 / meta 退详情卡、不进流**;**领域 tag 保留在流**(本轮落地反转 §4.5)、**不另上域色**(段已有色温,再加域色会打架)。
- **随记照记、不降权**(可能是用户灵感,不判断内容)。

### C.2 wizard preview（已有双预览，本次小改）

- preview **已同时画 大卡详情 + 流 item**(`CalendarBulletPreview`),随配置动态反馈 —— **所见即所得,不要拖拽**。
- 字段角色 **主/副 → 标题①/标题②**(信息 = 详情 meta、隐藏);标题①② = 流 item 那两格、大卡标题行同源(单一真值)。
- preview 加 **`↻ 重新生成`**:快捷 `更简单` / `内容更多` / `调整…`(自由说,如「去掉地点、加心情」)→ design-agent 重出 schema → 双预览刷新。**用自然语言把卡调到满意,不摊字段表单**(替代"手动加字段 / 字段 CRUD")。

### C.3 design 要出

- item「标题行」排版:`时间 · emoji · 标题① · 标题② · 领域 tag` 的间距 / 分隔符 / 截断;轻日子、密日子都成立。
- wizard preview 版面:**大卡 + 流 item 并排(或上下)** + 字段角色行(标题①/②/信息/隐藏)+ **`↻ 重新生成`** 控件(按钮 + 三快捷 + 自由输入)。
- 决策(design's call):标题①② 的分隔符(`·` / 空格 / 竖线)、emoji 与领域 tag 的位置、重新生成控件形态。

## §D · 颜色逻辑收敛（2026-06 定）

> 起因:流按「领域」上小色点、库按「per-skill accent」给容器染底 —— **两套含义、颜色太多**。**收敛成一条**。真值:[§5.1](../05-design-system.md) + [§8.3](../08-domain-system.md)。

- **颜色 = 「生活领域」单一轴**(8 域:工作/学习/健康/运动/社交/娱乐/生活/灵感),一套 8 色板。
- **退役 per-skill accent 选色**:卡片 / 库容器**本体单色**(neutral surface + 软阴影),**不再按技能 accent 染底**;wizard 去掉 7 色选色。
- **颜色只以一颗「领域小色点」出现在记录上**(流 item 右对齐小点;详情 hero 旁可点 domain chip)。`domain==null` 不显点。
- **技能 / 类目身份 = emoji + 名字**,不靠色。**库容器单色 + emoji**(技能跨多领域、没有单一域色,故不给容器上色)。
- **design 要打中**:整屏**克制**,颜色稀少 —— 只有那颗域点跳出来;别让 8 个域色 + 卡底色混成一锅花。
- 决策(design's call):域点的大小 / 位置 / 是否带 2 字领域名(紧时退纯点);8 域色的最终色值(可在 §5 的 8 槽基础上微调,但**保持低饱和、不刺眼**)。
