## 7.0 定位：一层「温柔」的游戏皮，服务"随手记"

把助理包进一层**轻**游戏皮,核心难点 = **让用户养成"随手记"的习惯**。本章管这层的**成果物侧**:把你的真实活动
结晶成 `completion_event` → 长成一座**周岛**(可晒的成果)。情感陪伴侧(球球)在 [§9](09-pet.md)。

**气质(护栏,两块 spec 共享):默认温柔** —— 无连胜 / 无愧疚 / 无进度条强迫 / 无排行榜;成长**只产外观、绝不锁功能**
(核心动作永远可用);奖励**只增不减、不付费抽、无 FOMO**。竞争元素(排行榜/攀比)依赖多用户社交,**后置**。

## 7.1 统一闭环（本设计的精髓）

```
个性化技能(无限)
   → agent 理解、grounded 生成「结构化任务」(L1,每条按内容带一个 domain)
   → 你在某领域做了「合格活动」:完成任务 / 录一条结构化记录 / 机会型实体创建
   → 填该领域的「日环」(每领域 ≤2/天,§8.4) ── 裸随记不计
   → 一条 completion_event(中心货币, append-only,带 domain)
   → 扇出三个 payoff:
        ① 长「周岛」对应**生活领域(domain)**区的元素   ← 本章 §7.4
        ② 掉「装饰物」(球球 inventory)                ← §9 宠物
        ③ 推「里程碑」                                ← §9 宠物
```

**为什么这套成立:**
- **岛长在「领域里的真实活动」上,但有日上限**(§8.4):一次合格活动 = 完成一条派生任务,**或**录一条该领域的结构化
  记录(记账/跑步/读书/勾 todo/事件赴约),**或**机会型一级实体创建。每条 → 填该领域日环 1 格 → 发 `completion_event` → 长岛/掉装饰。
  - **防刷靠「每领域 ≤2/天」封顶**(改自早期"只有任务完成才长岛"):录 100 笔记账 ≠ 100 元素,封顶 2。日上限既给真实活动回报,又堵刷量。
  - **唯一不自动计的是裸随记**(守"闪念不都长岛"):随手随记只进 L0;**灵感领域只由「升华一篇」任务**填(攒够 → synthesis,接 §6)。
  - **机会型一级实体(见 §7.3.2)**:`contact`(社交)/ `external_ref`(按内容)**创建即填 1 格**(`source=opportunistic`),同吃 2/天上限;`event` 走反应式任务。
- **有界 art 在「生活领域(domain)」层兜底**:技能无限,但岛只按 **8 个固定 domain**(工作/学习/健康/运动/社交/
  娱乐/生活/灵感)分区 → **千人千岛(做成什么不同)+ 美术只画 8 套区**。domain **按内容打**(见 [§8](08-domain-system.md))。
- `completion_event` 是 **L1→消费者** 的唯一缝(由**合格活动**产生,封顶 2/领域/天),append-only →
  幂等、可重算/回放/审计。
- **岛 ⊥ 宠物(解耦,本设计拆两 spec 的依据)**:两者都只**订阅 `completion_event`**,互不调用、可各自 ship/拔
  (本章 §7 = 岛模块按 domain 渲染;[§9](09-pet.md) = 宠物模块按 完成+里程碑 掉装饰/换装)。先做岛后做宠物、或反之都行。

## 7.3 任务体系（L1）

**今日待完成 = grounded、数据驱动、条件触发的「闭环 nudge」**。**不是必须清空的清单** —— 每条独立、可选,
**不要求全完成、不做进度条、不愧疚**。完成**任意一条** = 一个闭环 = 给奖励(只加不罚)。

**每条任务 = 一个带「触发条件 + 完成判定」的对象:**
```
{ 标题, grounded 理由, 触发条件, 完成判定(对你的数据求值的谓词),
  domain(按内容打,8 选 1,§8), tier(简单/中等/高难), reward_hint }
```

**两类来源(daily-gen agent 每天评估一次):**
1. **基底项(直给、稳定)** —— 未完成 todo、今日日程。直接列,勾选即完成。稳定锚,避免"满屏新花样"。
2. **触发/推断项(条件命中才出现,因人因日而异)** —— 监测你的数据模式:
   - **阈值型**:攒了 10 条灵感 → "合成/升华成一条更深的想法"(判定 = 出现 synthesis 资产);读了 150 页
     → "录一条读书感想"。
   - **缺口型**:今天记了消费没记三餐 → "补今天三餐"(判定 = 三个时段各一笔);有跑步计划今天没记 → "记跑步"。

**生成机制:** **daily-gen agent**(结构化输出,跟 flash dispatcher 同形)每天跑一次:拉基底项(直查 todo/event)
+ 评估一组**触发规则**(对累计计数器 + 技能字典求值)→ 命中的生成触发项,每条自带 `completion_predicate`。
**缓存当天、可手动刷新**;别每次进页面都重算。**grounded 铁律**:只把你**已表达的需求/习惯/技能**翻成今日可
闭环的事,**绝不发明**。触发规则的**具体数值(10?150?)= 调参,后定**。

**完成判定:** 基底项(todo/日程)勾选即落库;触发项做了对应动作(建了那条 idea 合并 / 读书感想 / 三餐)由其
`completion_predicate` 对数据求值。**任务完成 → 落 `completion_event`**;此外**该领域的结构化记录**也按 per-domain 日环计入(封顶 2/天,§8.4)——两条路都长岛,唯裸随记不计(见 §7.1)。

### 7.3.1 配额 / 难度 / 岛计数 → 口径见 [§8.4](08-domain-system.md#84-domain--每日任务派生任务体系从-per-skill-预算--per-domain-日环)

> 配额模型**已从 per-skill 预算改为 per-domain「日环」**(旧 7/3/1 折叠进 [§8.4](08-domain-system.md))。这里只留岛侧的不变量:

- **岛计数 = 绝对:每填 1 格日环 → +1 个 icon**(**不**用完成率)。**越多越茂盛**。
- **每领域 ≤2/天**计入岛(§8.4)= 反刷阀,也是「贫瘠↔丰满」诚实信号的守门(做得少=岛偏空,不拉平,值得晒 / 对比上周)。
- **难度三档 ≈ cadence**:`简单`=每日 routine / `中等`=每周小目标 / `高难`=每周·**领域级**(把本周某领域升华一篇,§8.4)。**高难完成 → 稀有 / 更大的地标 icon**。
- **UX:** 活跃领域 / 技能一多 → **每日视图必须 curate**(只推今天最相关的一小撮,其余可展开),别 dump 全部。
- 千人千岛**差异来自**:做成的总量(贫瘠↔丰满)+ 领域组合 + 稀有地标 + 装饰。具体数字(每领域 2 / 活跃判定 / 升级阈值 / domain→element 映射 / landmark)= **进 [§10 游戏配置](10-game-config.md)**(岛实现时按其 schema 落,别硬编)。

### 7.3.2 各资产类型的任务来源（常驻 vs 自定义）

> 计入岛的口径统一为 per-domain「日环」(≤2/天),见 [§8.4](08-domain-system.md#84-domain--每日任务派生任务体系从-per-skill-预算--per-domain-日环)。下表列各类型如何**填环**。

| 类型 | 任务来源 | domain（按内容,[§8](08-domain-system.md)） | 如何填日环 |
|---|---|---|---|
| **自定义习惯技能**(记账/跑步/读书/工作记录…,≤9) | grounded daily-gen 派生该领域任务 | 技能 prior(记账→生活、跑步→运动、读书→学习…),agent 可按内容微调 | **完成任务 或 直接录一条结构化记录**(≤2/领域/天) |
| **代办 todo**(常驻) | **base 直给项**:完成你已有的待办 | **按内容**(交报告→工作、买菜→生活、健身→健康) | **勾完成**(按其 domain 填环) |
| **事件 event**(常驻) | **反应式任务**:今日日程→「确认/赴约」、临近会议→「准备」 | **按内容**(开会→工作、聚餐→社交、看演出→娱乐) | 赴约 / 完成反应式任务(按其 domain 填环) |
| **随记**(常驻) | **轻 nudge + 阈值触发**:「攒了 N 条想法→升华一篇」(接 §6) | **灵感** | **裸随记不计**;只「升华一篇」任务填灵感环 |
| **名片 contact**(常驻) | **机会型,无任务** | 社交 | **创建即填 1 格**(社交) |
| **外部 external_ref**(常驻) | **机会型,无任务** | **按内容**(同步到工作工具→工作) | **创建即填 1 格** |

- **结构化记录直接填环(§8.4 新口径)**:习惯技能 / todo / event 的**真实记录或完成**按其 domain 填日环,**每领域封顶 2/天**(反刷)。
- **机会型一级实体(contact / external)**:创建即发 `completion_event(source=opportunistic)` 填 1 格(纯机会、无法预告),同吃 2/天上限。
- **唯一不自动填环的是裸随记**(守「闪念不都长岛」,§7.1)——灵感环只由「升华一篇」填。

## 7.4 周岛（新核心 · 成果物）

你这周**填进日环的格子**(任务完成 + 结构化记录 + 机会型实体,≤2/领域/天,见 [§8.4](08-domain-system.md#84-domain--每日任务派生任务体系从-per-skill-预算--per-domain-日环);裸随记不计,§7.1),**程序化渲染成一座俯视小岛**(`worldgen`),周末出**分享卡**朝外晒。

- **领域驱动 + 固定元素**:**日环每填 1 格** → 按其 **domain(生活领域)** 长对应**有界固定**的"区/城"元素(裸随记不长)。
  | 生活领域 | 岛区元素 |
  |---|---|
  | 工作 | 工坊 / 塔楼 |
  | 学习 | 灯塔 / 图书馆 |
  | 健康 | 果园 / 花园 |
  | 运动 | 步道 / 球场 |
  | 社交 | 小屋 / 篝火 |
  | 娱乐 | 帐篷 / 营火 |
  | 生活 | 集市 / 民居 |
  | 灵感 | 观星台 / 思维树 |
  无 domain / `null` **不长岛**(或极小底色)。
- **画布 = 固定 12×12**(已拍板;概念用 14×14、10×10 偏紧)。**画布固定、不随任务数变** —— 因为下面的
  「合并/升级」让岛靠**加密 + 升级**长茂盛,而非加格子。
- **丰满度 = 绝对计数**:日环每填 1 格 → +1 个元素(**越多越茂盛**);**贫瘠/丰满有别 = 诚实信号**(§7.3.1 / §8.4)。
  每领域 2/天封顶、不算分母。**同周数据 → 同岛(seed 固定)** → 可复现、可对比上周。
- **合并/升级(两层,已拍板):**
  1. **视觉自动升级**:同 domain 攒够 → 小元素**自动合并升级**成大结构(`麦束→农田→谷仓`、`树苗→树林→古树`);
     tier 决定起点(简单=小元素、中等=更饱满、**高难=稀有地标**)。这给"看它长大"的成长感,**也防溢出**
     (完成不各占一格,而是升级现有区域 → 12×12 够用)。
  2. **内容层(接 [§6](06-synthesis-report.md))**:**高难任务 = "把 N 条 X 合成一篇"** → 完成产出一份**报告
     artifact** + 在岛上栽一个**稀有地标**。"升级"在内容上 = 合成报告、在岛上 = 地标,**同一动作**。
- **满载**(max 用户完成所有任务):岛"满"= **~⅔ 画布被茂密/升级区域 + 多地标占住,留出 shore 才像"岛"**
  (**不**100% 铺满 —— 那样像砖块、难看)。
- **一套精灵全员通用 → 美术成本低、千人千岛**(差异来自 领域组合 + 升级程度 + 稀有地标 + 装饰,**不是**记录量)。
- **本周岛全周 live 生长**:进「我的岛」就看到本周岛随**任务完成**实时变茂盛/升级(钩子);**周末定格成分享卡**
  (岛 + 球球在岛边 + 计数:完成数/连续天/解锁装饰数。**不显 LV**)。
- 住「我的岛」tab;**历史周岛**可往周回看。**月岛 / 年岛 = 后置**。美术风格(俯视像素 / iso / 扁平)走单独 design doc。

## 7.9 数据模型增量

> 完整列定义见 [§2](02-data-model.md);HTTP 接口见 [§3.15 engagement 接口](03-api-reference.md);此处给契约。

- **`skills` 加 `domain`**(String(20), nullable;**默认 prior**,§8)。结构化技能有稳定 prior(跑步→运动);
  常驻/随记为 null → 由 agent 按内容 per-task 打。
- **`daily_plans`(L1)**:`{id, user_id, day, budget(N), items(JSON:[{title, reason, trigger, completion_predicate,
  domain(8 选 1,按内容), tier(简单|中等|高难), skill_ref, status}]), generated_at}`。一天一行,缓存当天。
- **`completion_events`(append-only,中心货币)**:`{id, user_id, domain, tier, source(task|record|opportunistic), ref, ts}`。
  由**合格活动**产生(§8.4):`task`=任务完成(含 event 反应式)、`record`=该领域结构化记录(记账/跑步/读书/勾 todo)、
  `opportunistic`=机会型一级实体(contact/external_ref)。**每领域封顶 2/天**(去重 / 限流在写入处保证);**裸随记不发**。L2 / 岛 / 里程碑只读消费,**绝不回头解读语义**。
- **`weekly_islands`(快照/历史)**:`{id, user_id, week_start, snapshot(JSON: domain→{count, tier 分布}), seed, card_image?, created_at}`。
  当前周可实时算;历史周存快照供回看/对比/分享。
- **宠物侧表**(`mascot` / `mascot_inventory` / `milestones` / `cosmetic_catalog`)→ 见 **[§9.5](09-pet.md)**(本章只产 `completion_event`,不持宠物状态)。

## 7.11 盈利与护栏（freemium）

> **本节 = 护栏原则;具体定价模型 + 成本账见 [§12 商业模式](12-business-model.md)(pending)。**

- **计量两样**:**闪念捕捉数** + **自定义技能数**,付费扩容。**premium 装饰 / 形态** = 干净付费面(只卖外观,详见 [§9.7](09-pet.md))。
- **安全阀**:① 免费捕捉额度设在习惯人群日用量之上(养成期不撞墙);② **撞墙绝不吞用户的话**(输入照存/排队,
  只软 upsell);③ 激励返点(完成多的投入用户挣更多 headroom)。**严禁刷量换额度的逼迫式 grind**。
- 具体额度 = 调参,后定(提案值见 [§12.2](12-business-model.md):捕捉 30/天 · chat 300/月 · 报告 ~4/月 · 洞察 1/周)。
