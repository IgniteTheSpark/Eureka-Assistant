08
CASE 1

日历页面如何被重做三次

V1 · 传统顶部日期

第一版很快能看懂，但也很草率：像普通日程列表，缺少 UReka 自己的节奏和记忆感。

V2 · 参考 Timepage

第二版把滚动和动效做出来了，视觉方向更强；但内容展示仍然不贴近日常记录的预期。

V3 · Agent 反复讨论后

最终版本把日程、非日程、分类资产和当天语境放在一起，交互才开始服务内容。

11
HARDWARE

硬件戒指接入：把闪念按钮戴在手上

这页重点不是卡片设备，而是“按一下就记录”的入口设计：让记录动作从打开 App 变成一个肌肉记忆。

入口变轻用户不需要找手机、不需要先决定记到哪里。
动作明确戒指按钮把“闪念来了”变成稳定的物理触发。
Agent 接手后续再完成 ASR、理解、分类、资产写入。
产品判断硬件价值来自降低记录门槛，而不是多一个硬件卖点。
戒指按下→录音触发→ASR→理解归类→资产
