# 7·17 戒指模型与触控区设计

## 目标

把设计师导出的 `戒指_7_17.glb` 作为 landing page 唯一的实时戒指模型，并在「触」章节对新增的电容触控区做克制的局部强调。同时消除静态占位图与 WebGL 模型交叉显示造成的戒指重叠、透视错位和“变形”观感。

## 模型结论

- 新模型保留现有外壳、内环、电路、金属触点与整体尺寸。
- 新增的电容触控区位于主戒指网格中，使用独立材质 `材质.005`。
- 页面不重建触控区几何，不改变设计师提供的拓扑、UV 或默认材质外观。
- 新文件使用独立 URL `/ring/ring-capacitive-7-17.glb`，避免浏览器继续命中旧 GLB 缓存。

## 视觉行为

- 默认、Flash、Vibe、「说」和「感」章节：触控区保持模型原始深色材质，不额外发光。
- 「触」章节：仅 `材质.005` 逐渐增加很轻的紫色 emissive 与环境反射，让用户意识到该区域可触控，但不把它做成灯带。
- 离开「触」章节：材质平滑恢复原始颜色、粗糙度、金属度、环境反射和 emissive。
- 外壳颜色与现有整页滚动旅程继续由 `exteriorMaterials` 驱动，触控区不加入外壳材质集合。

## 单一渲染源

- WebGL 不可用、加载失败或尚未完成首帧时，展示静态 PNG 兜底。
- WebGL 首帧 ready 后，静态 PNG 从 DOM 卸载，不参与后续章节切换。
- 后续滚动、颜色切换和章节切换始终只存在一枚实时戒指，避免两个不同透视的戒指通过透明度叠加。

## 验证

- 配置测试锁定新 GLB URL 与触控材质名。
- 模型准备测试验证 `材质.005` 会被独立收集，且不会被误归类为外壳。
- Stage 测试验证 WebGL ready 后静态 poster 被卸载；WebGL 不可用时 poster 仍然存在。
- 运行 landing 聚焦测试、类型检查和生产构建。
