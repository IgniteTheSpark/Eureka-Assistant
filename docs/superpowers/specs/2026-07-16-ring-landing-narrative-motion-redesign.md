# Eureka Ring Landing Page 叙事与滚动动线重构设计

## 目标

把当前 Landing Page 从“先展示模式入口，再重复解释功能”的结构，重构为一条由同一枚全屏 3D 戒指贯穿的产品故事。页面应优先讲清楚 Eureka Ring 如何捕捉想法、如何连接电脑工作、如何形成完整的个人智能入口，而不是继续展开真机 Demo 或内部硬件结构。

本次重构必须保留现有全屏固定的 3D 戒指舞台。场景图片、系统图和功能文字都是戒指的滚动落点，不能用静态图片替代跨屏滚动的戒指交互。

## 页面叙事顺序

页面调整为以下七个章节：

1. Hero：首页主张与产品第一印象。
2. 双模式过渡：用一小段文案建立 Flash 与 Vibe 两种体验，不放场景图，不提供模式入口或链接。
3. Flash：解释从自然表达、理解意图、调用 Skills 到形成资产的闭环，并在标题下方呈现驾驶场景图。
4. Vibe：解释语音如何进入当前桌面工具，并在标题下方呈现办公室场景图。Codex 与钉钉明确标注为示例软件，而非完整支持列表。
5. 系统：解释 Eureka Ring、本地桌面连接、个人智能层、工具与资产之间的关系。
6. 说、触、感：用三段独立内容解释产品的输入、手势和反馈能力。
7. 内测 CTA：保留二维码占位与进群说明。

删除以下内容：

- 当前双场景模式选择大图及其链接交互。
- “一枚戒指背后的五件事”章节。其内容与 Flash、Vibe 和系统章节重复。
- 内部结构和三个戒指视图章节。本阶段不展开具体结构讨论。

## 全屏戒指舞台

### 不变项

- `LivingRingStage` 继续固定在整个视口，覆盖完整页面滚动周期。
- 戒指继续随滚动更新位置、旋转、尺度、材质颜色和光照。
- 戒指保持 `pointer-events: none`，不阻碍页面阅读和操作。
- 3D 场景继续懒加载，并保留静态海报作为加载前回退。
- `prefers-reduced-motion` 下取消跳动、翻滚和连续自转，只保留章节间的静态位置与淡入淡出。

### 新的滚动锚点

当前每个章节只有一个戒指停靠点，无法表现“从标题滚入场景图”。重构后，滚动映射支持章节内部锚点：

- `hero`
- `modes`
- `flash-intro`
- `flash-scene`
- `vibe-intro`
- `vibe-scene`
- `system-start`
- `system-end`
- `speak`
- `touch`
- `feel`
- `community`

首页滚动测量从只扫描 `section[data-ring-chapter]` 改为扫描所有具备 `data-ring-chapter` 的锚点元素。每个锚点仍映射到一个确定的 3D 姿态，锚点之间使用平滑插值。

### Flash 动线

- 戒指进入 Flash 章节时位于“录音只是开始”左侧，尺度中等，不遮挡标题。
- 随页面继续向下，戒指向驾驶场景图中人物佩戴戒指的手部移动，同时逐步缩小并调整透视角度。
- 接近照片中的戒指位置时，3D 戒指淡出，避免与图片中已经存在的戒指出现双重叠影。
- 离开场景图后，3D 戒指重新淡入并进入下一个章节。

### Vibe 动线

- 戒指从 Flash 场景退出后跨屏移动到 Vibe 标题一侧，颜色逐步过渡到蓝紫色。
- 随滚动进入办公室场景图，戒指对准人物手部位置并在重合前淡出。
- Codex、钉钉说明位于场景图附近，但不做可点击入口；旁边明确显示“示例软件，支持范围不限于此”。

### 系统动线

- 戒指缩小到当前系统章节尺寸的约 40%，保持在视口中轴线附近。
- 系统章节背景增加一条纵向青绿色 Neon 轨道。轨道使用低透明度核心线、柔和外发光和局部亮点，不使用大面积高强度模糊。
- 随页面滚动，系统步骤从下方向上经过戒指，形成戒指沿中轴线下行的视觉关系。
- 戒指和 Neon 轨道处于正文之后，任何宽度下都不得遮挡标题或步骤文字。

### 说、触、感动线

- 戒指移动到视口右侧的安全轨道，正文占据左侧和中部。
- “说”“触”“感”各自形成一个滚动锚点。
- 从一个锚点进入下一个锚点时，戒指完成一次受滚动控制的翻滚。翻滚只由滚动进度驱动，不增加独立的永久动画循环。
- 右侧轨道为戒指预留固定安全区；桌面端文字最大宽度不超过页面的 65%，移动端戒指移到内容上方并停止翻滚。

## 场景图布局

### Flash 场景图

- 驾驶图放在 Flash 标题、说明和处理链路之后，成为本章节的主要视觉画面。
- 图片使用横向宽幅裁切，保留人物面部、手部、驾驶环境和语音提示；移除二维码相关区域。
- Flash 示例资产位于图片之后，避免先解释示例再进入使用场景。

### Vibe 场景图

- 办公室图放在 Vibe 标题和说明之后。
- 图片保留人物手部、Codex 屏幕、咖啡和语音提示，移除二维码相关区域。
- Codex 和钉钉示例以文字标注或简洁列表附在场景图之后，不使用链接箭头或按钮式样。

### 响应式规则

- 桌面端场景图保持宽幅，戒指可以精确接近手部目标。
- 平板端调整裁切位置，戒指只接近手部区域，不追求像素级重合。
- 移动端取消戒指与照片手部的精确交接，改为进入场景图前缩小淡出，避免不同裁切比例造成明显错位。

## “说、触、感”内容

### 说

自然语音输入。用户无需寻找另一块屏幕，通过戒指直接表达想法或指令。

### 触

支持 7 种不同的手势交互。此章节只表达能力数量和手势交互的价值，暂不列出每个手势与具体动作映射。

### 感

支持 3 种不同的震动反馈。此章节只表达反馈有三种可区分的节奏或模式，暂不定义其具体业务含义。

## 视觉层级

- Hero 保持明亮、克制的产品展示感。
- Flash 使用偏暖的红橙色材质过渡，与“捕捉、生成”建立关联。
- Vibe 使用深蓝和蓝紫色，强调桌面工具与控制感。
- 系统使用浅青绿色背景与 Neon 中轴线，表达持续连接。
- 说、触、感使用较深的品牌色背景，为右侧翻滚的戒指提供对比。
- 章节可以拥有不同色彩世界，但字体、留白、戒指材质和滚动节奏必须保持同一产品语言。

## 性能约束

- 不新增第二个 WebGL Canvas，也不复制 3D 模型。
- 所有戒指运动继续在同一个固定舞台中完成。
- 滚动运动只修改 transform、姿态和材质参数，不读取高频 DOM 布局。
- 页面尺寸变化或 ScrollTrigger 刷新时才重新计算锚点位置。
- 场景图使用 WebP 并懒加载；进入相邻章节前可提前解码，但不一次性加载全部大图。
- Neon 效果由有限层数的 CSS 渐变与阴影构成，不使用持续变化的大面积滤镜。
- 翻滚、跳动和色彩变化均由滚动进度控制，不创建额外的常驻动画循环。

## 可访问性

- 场景图片提供描述使用情境的中文替代文本。
- 说、触、感使用语义化标题，不只依赖单个大字表达含义。
- 全屏戒指为装饰性视觉，不进入键盘焦点顺序。
- `prefers-reduced-motion` 下取消场景交接、跳动和翻滚，确保内容仍完整可读。
- 文字和背景对比度满足 WCAG AA，尤其是 Vibe 深色章节与 Neon 系统章节。

## 验收标准

- 页面中不再出现可进入 Flash 或 Vibe 真机 Demo 的链接。
- 双模式过渡章节不再包含两张场景图。
- 驾驶场景图只出现在 Flash 章节，办公室场景图只出现在 Vibe 章节。
- 全屏戒指从 Hero 持续存在到内测 CTA，并按新锚点跨章节移动。
- Flash 与 Vibe 桌面端能看到戒指向人物手部移动并淡出的过程。
- 系统章节戒指明显更小，并沿中轴 Neon 轨道滚动，不遮挡正文。
- “一枚戒指背后的五件事”和内部结构章节被移除。
- 页面新增“说、触、感”，分别说明语音输入、7 种手势和 3 种震动反馈。
- Codex 与钉钉被明确标注为示例软件。
- 桌面、平板、移动端没有标题溢出、图片错位或戒指遮挡正文。
- 页面静置后 DOM、监听器和文档数量保持稳定，不重新引入此前的递归加载或持续内存增长。

## 测试范围

- 内容测试：章节顺序、删除项、说触感文案、示例软件说明。
- 滚动状态测试：新锚点顺序、章节姿态、淡出交接、系统缩放、右侧翻滚。
- 页面测试：双模式区域无链接，场景图位于正确章节。
- 响应式浏览器检查：1440×900、1024×768、390×844。
- 性能回归：浏览器静置前后 DOM 节点、监听器、文档数和 JS Heap 不持续增长。
- 完整单元测试与生产构建。
