# Eureka Ring 双行生态、说触感与系统总结设计

## 目标

重构 Landing Page 的后半段，让页面从两个使用模式自然过渡到戒指本身的交互能力，再以品牌宣言和系统闭环完成总结。此次设计需要：

- 将 Vibe 的软件生态改成双行 Logo Loop，并减少解释性小字。
- 将“说、触、感”从静态信息表升级为三个连续的戒指交互演示。
- 将“它不是一个缩小的语音助手”移动到“说、触、感”之后，承担品牌总结与轻量系统说明。
- 保留贯穿全页的唯一 3D 戒指，不重置模型，不用另一个视觉替代。
- 压缩重复内容和无意义留白，同时维持产品展示所需的呼吸感。

## 页面叙事顺序

后半段顺序调整为：

```text
Flash
  → Vibe
  → 说 / 触 / 感
  → 品牌宣言与系统闭环
  → 内测 CTA
```

“说、触、感”负责展示戒指本身如何输入、控制和反馈；系统总结段负责解释这些能力如何通过电脑或手机连接个人智能、应用与资产。系统总结不再出现在功能演示之前。

## Vibe：双行软件生态

### 内容

Logo Loop 使用两条持续滚动的轨道，展示 Codex、钉钉、VS Code、Cursor、GitHub、Notion、Figma、Slack、Chrome 等办公与效率软件。第二行以 `and even more` 收尾。

底部只保留一句说明：

> 以上为常见办公与效率软件示例，连接范围仍在持续扩展。

不再解释 Codex 与钉钉用于当前演示，也不再使用“封闭兼容清单”等偏工程说明。

### 运动

- 第一行由右向左，第二行由左向右。
- 两行使用不同周期，避免形成机械同步；桌面端周期分别为 34 秒与 42 秒。
- 悬停任意一行时，两行同时暂停；移开后继续滚动。
- 边缘使用渐隐遮罩，避免 Logo 突然切断。
- Reduced Motion 下取消自动滚动，两行自然换行并完整展示。

### Logo 资源接口

`AppLogoItem` 增加可选 `src` 与 `alt`。真实 Logo 以透明背景 SVG 或 PNG 放入 `public/logos/`；在用户提供品牌资源前，保留当前内置标识作为 fallback。替换资源不改变轨道结构与动画逻辑。

## 说、触、感：三个连续交互场景

### 总体结构

该章节由一个总标题和三个 `min-height: 92svh` 的场景组成。每个场景左侧保留极简文字，右侧由全局 3D 戒指和与其对齐的 HTML 特效层完成演示。减少现有大面积空白，但不把内容压缩成普通卡片列表。

戒指从 Vibe 场景离开后进入“说”，随后以硬币式翻转进入“触”和“感”。每次章节切换完成约 180° 的 Y 轴翻转；滚动反向时动画自然反向，不发生跳帧或模型重置。

### 说

文案：

- 标题：`说`
- 能力：`自然语音输入`
- 说明：`想法或指令，直接说出来。`

效果：

- 戒指停在画面右侧展示位。
- 从戒指中心向外扩散三层语音波纹。
- 波纹使用不完全闭合的弧线而不是雷达圆环，以减少“扫描器”感。
- 波纹的振幅、透明度和间距随滚动进度增强；不读取真实麦克风。

### 触

文案：

- 标题：`触`
- 能力：`7 种手势交互`
- 说明：`同一个动作，在不同设备、应用和场景中承担不同含义。`

效果：

- 戒指从“说”完成一次 180° 硬币翻转。
- 七个已验证手势沿戒指外围依次出现：`单击`、`双击`、`三击`、`上滑`、`下滑`、`左滑`、`右滑`。
- 不展示长按，因为当前产品资料明确不将其作为稳定真机能力。
- 标签采用不同半径和轻微错位，滚动时按顺时针顺序点亮；不组成密集的完整圆盘。

### 感

文案：

- 标题：`感`
- 能力：`3 种震动反馈`
- 说明：`不用查看屏幕，也能感知状态与结果。`

效果：

- 戒指再次完成一次 180° 硬币翻转。
- 三种已验证震动以不同运动语言表达：
  - `强力`：一次短促、明显的收缩与扩散。
  - `持续`：等幅、稳定重复的细线脉冲。
  - `渐变`：由弱到强的三段递进波纹。
- 动效只作为抽象表达，不宣称精确模拟物理震感。

### 特效组件边界

新增独立 `SenseRingEffects`，渲染在 `LivingRingStage` 内，与全局戒指共享相同的固定坐标和 CSS transform。父级 `data-ring-chapter` 决定显示 `speak`、`touch` 或 `feel` 特效，避免在页面中复制戒指。

所有特效仅动画 `transform` 与 `opacity`。Reduced Motion 下：

- 戒指不执行连续翻转。
- 语音波纹显示为静态弧线。
- 七种手势标签一次性显示。
- 三种震动显示为三个静态图形签名。

## 品牌宣言与系统闭环

### 定位

该章节是产品总结，不再重复介绍四个功能模块。整体使用 `150svh`，内部使用 `100svh` 的 sticky 场景，通过滚动逐步完成品牌宣言和系统结构展开。

### 品牌宣言

主标题：

> 不是把助手缩小。  
> 是让智能，始终在手边。

辅助说明：

> Eureka Ring 通过电脑或手机连接个人智能，让语音、手势和触觉反馈进入用户正在使用的应用与设备。

### 系统闭环

系统不是“戒指 → 桌面”的单向链路，而是通过电脑或手机形成双向闭环：

```text
                     个人智能
             上下文 · Agent · Skills
                        ↓
说 / 触 → Eureka Ring → 电脑或手机 → 应用、AI 与资产
             ↑                           ↓
             └──────── 触觉反馈 ─────────┘
```

结构含义：

- `Eureka Ring`：采集语音和手势，并通过震动返回反馈。
- `电脑或手机`：必要的连接、计算和当前环境感知层，不限定为某一种终端。
- `个人智能`：负责上下文、Agent、Skills 与意图理解。
- `应用、AI 与资产`：执行操作，并形成消息、日程、联系人、文档、随记等结果。
- `触觉反馈`：将完成、确认和状态变化安静地带回戒指。

### 视觉展开

- 戒指从“感”完成最后一次翻转后进入页面中轴并停稳。
- 电脑和手机作为一个“连接设备”节点，从左右两侧向戒指靠近。
- “个人智能”节点从上方出现，“应用、AI 与资产”从下方出现。
- 光路依次连接节点，最后从结果端回到戒指，形成不完全闭合的环。
- 不使用四张大卡片，不保留 `01–04` 装饰编号，也不再罗列重复的技术能力胶囊。

## 戒指路线

调整后后半段路线为：

```text
Vibe 右下场景
  → 说：右侧展示位
  → 触：右侧展示位，Y 轴 +180°
  → 感：右侧展示位，Y 轴再 +180°
  → 系统总结：移动到中轴并停稳
  → CTA：缩小并作为收尾视觉
```

`LANDING_RING_STOPS` 的叙事顺序同步调整为：

```text
vibe-exit → speak → touch → feel → system-start → system-end → community
```

“说、触、感”期间戒指保持完整可见；系统段只在中轴出现，不覆盖品牌宣言和节点文案。

## 响应式

### 桌面端

- 双行 Logo Loop 全宽穿过 Vibe 左下内容区。
- 说、触、感使用左文右戒指结构。
- 系统总结以中轴戒指和四向节点构成闭环。

### 移动端

- 双行 Logo Loop 维持双轨，但缩小间距和 Logo 尺寸，禁止扩大页面宽度。
- 说、触、感每段使用上文案、下戒指效果的纵向结构。
- 七种手势标签最多分两圈排布，保证不超出视口。
- 系统节点改为纵向：个人智能 → 戒指与连接设备 → 应用与资产，反馈弧在右侧回到戒指。

## 性能与可访问性

- 不新增动画库，继续使用 GSAP、ScrollTrigger 和 CSS transform。
- Logo Loop 使用纯 CSS transform；说、触、感的滚动状态由现有全局旅程驱动。
- 不动画布局属性，不创建第二个 WebGL Canvas，不增加新的 3D 模型实例。
- Logo 列表保留一个可访问主列表，动画副本设置 `aria-hidden="true"`。
- 动效文本使用真实 DOM 文本；装饰波纹和轨道对辅助技术隐藏。
- 页面在 `prefers-reduced-motion: reduce` 下完整可读、可操作。

## 测试与验收

- 单元测试验证双行 Logo Loop 的两个方向、可访问主列表和 `and even more`。
- 单元测试验证七种手势名称，不出现长按。
- 单元测试验证强力、持续、渐变三种震动签名。
- Journey 测试验证章节顺序、每段约 180° 翻转和系统中轴收束。
- LandingStory 测试验证“说、触、感”位于系统总结之前。
- 在 1440×900 与 390×844 中检查：无横向溢出、戒指不遮挡文字、特效与戒指对齐、Logo 轨道不撑宽布局。
- Reduced Motion 下检查内容完整显示且无持续动画。
- 最终运行全部 Vitest、TypeScript typecheck、生产构建与浏览器 console/network 检查。
