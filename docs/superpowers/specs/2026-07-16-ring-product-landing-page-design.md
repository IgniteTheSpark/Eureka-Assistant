# Eureka Ring 中文产品 Landing Page 设计

日期：2026-07-16  
状态：已批准，进入实现

## 1. 产品定位

Eureka Ring 当前处于内测阶段。Landing Page 同时面向展会访客、潜在用户、合作伙伴和投资人，但阅读顺序优先服务第一次接触产品的人：先理解产品价值与使用场景，再获得技术可信度，最后扫码加入内测群。

整站使用中文。`Flash Mode` 和 `Vibe Mode` 可以作为英文产品模式名保留，但首次出现时必须同时提供中文解释：

- Flash Mode｜闪念：把自然表达理解为意图，并形成可继续使用的资产。
- Vibe Mode｜随声操控：把语音与戒指动作交给当前正在使用的桌面工具。

## 2. 目标

- 把当前 Demo 首页扩展为完整的产品 Landing Page。
- 用连续滚动叙事解释产品、两种模式、工作原理、核心能力与硬件形态。
- 让同一个 3D 戒指贯穿整页，并成为章节切换的视觉线索。
- 允许戒指颜色、材质和内部光效随“智能状态”变化，但不暗示所有颜色均为量产配色。
- 保留现有连接戒指与 Flash/Vibe Demo 入口；本轮不重做两个 Mode 的真机操作页。
- 页面末尾提供明确的内测入群 CTA 和可替换的正式二维码展示位。

## 3. 非目标

- 不新增 Flash/Vibe 真机流程或修改 Ring Desktop 路由。
- 不增加未经确认的重量、续航、防水、材料或价格参数。
- 不把动态颜色描述为可售 SKU。
- 不使用生成的假二维码；正式二维码由用户后续提供并替换静态资源。
- 不增加新的远端服务、埋点系统、CMS 或营销表单。

## 4. 设计原则

### 4.1 戒指是连续的“活体导航物”

页面只维护一个全局 Ring Stage。戒指从 Hero 出发，随滚动在八个章节的锚点之间连续插值，而不是在每个 Section 重新渲染一枚戒指。

变化维度包括：

- `position`：在文案、场景照片、系统图和 CTA 之间停靠。
- `rotation`：展示外壳、内环和电路的不同角度。
- `scale`：Hero 强展示、内容章节退让、硬件章节重新放大。
- `color`：作为智能状态变化，不等同于产品配色。
- `roughness / metalness / envMapIntensity`：让同一颜色呈现不同实体材质。
- `pulse`：驱动内侧电路或局部光源呼吸。
- `hop`：章节交界处的一次受控跃迁，不做持续弹跳。

### 4.2 动效必须有语义

- 旋转：切换观察对象或进入新能力时使用。
- 轻微悬浮：只用于章节停靠后的待机状态。
- 跳动：只在进入 Flash、Vibe 和 CTA 等关键章节时发生一次，幅度不超过视觉直径的 4%。
- 电路呼吸：用于 Agent/Skills 与系统连接章节。
- 颜色过渡：在相邻章节间连续插值，不突然切色，不做无意义彩虹循环。
- `prefers-reduced-motion` 下取消连续旋转、跳动和视差，仅保留即时位置、缩放与颜色切换。

### 4.3 中文内容先讲价值，再讲机制

正文避免堆积“AI”“Agent”等术语。每个技术名词都必须解释其对用户意味着什么。页面不使用重复的小型图标卡片网格，也不在每个章节重复相同的编号眉题。

## 5. 页面叙事

### 5.1 Hero：智能，触手可及

主标题：

> 智能，触手可及。

副文案：

> 一枚连接个人智能的戒指。捕捉转瞬即逝的想法，也让声音成为操作电脑的方式。

操作入口：

- `向下探索`
- `申请内测`

Ring 状态：深石墨实体材质，右侧大尺寸悬浮；缓慢自转但不完整循环。用户向下滚动时开始翻转，露出内环。

### 5.2 模式选择：一枚戒指，两种智能体验

保留当前左右人物场景和中间产品跑道，中文化所有文案。用户不需要先连接戒指即可进入模式页。

Flash：

> 说出一个想法，让 Eureka 理解它，并把它变成可以继续使用的内容。

Vibe：

> 对正在使用的工具直接说话，让声音成为下一种电脑交互方式。

Ring 状态：停靠中央跑道；悬浮 Flash 时向左手部移动并过渡为琥珀/珊瑚能量态，悬浮 Vibe 时向右手部移动并过渡为钴蓝/紫罗兰能量态。离开后回到中央。

### 5.3 Flash：想法不再停留在录音里

标题：

> 录音只是开始。

正文：

> Eureka 会理解你真正想完成的事情，调用合适的能力，再把一句自然表达变成下一步可以继续使用的资产。

展示一条会随滚动展开的语义链路：

```text
一句自然表达 → 理解意图 → 调用 Skills → 形成资产
```

轮换示例：

- “提醒我明天下午联系 Kevin” → 待办与联系人
- “记一下这个产品方向” → 结构化随记
- “安排周五四点的会议” → 日程
- “把刚才的想法整理成项目计划” → 文档资产

Ring 状态：从人物手部回到页面右侧，做一次轻微上跃；外壳从暖琥珀过渡到珊瑚色，内侧电路以低频脉冲回应语义链路。

### 5.4 Vibe：让声音进入正在使用的工具

标题：

> 不需要切换到另一块屏幕。

正文：

> 对着戒指说话，指令直接进入你正在工作的地方。Eureka 负责连接语音、戒指动作与当前桌面应用。

主要目标：

- Codex：描述需求、执行计划、修改项目。
- 钉钉：输入内容、导航界面、完成日常沟通。

Ring 状态：横向翻转到左侧，色彩从珊瑚连续穿过洋红进入钴蓝与紫罗兰；出现一次短促的方向性脉冲，表达指令进入应用。

### 5.5 工作原理：从手指到个人智能

标题：

> 它不是一个缩小的语音助手。

正文：

> Eureka Ring 是人与个人智能之间持续在线的入口。

系统链路：

```text
Eureka Ring
语音与动作
      ↓
本地桌面连接
录音、识别、应用感知
      ↓
个人智能层
Agent、Skills、上下文
      ↓
工具与资产
Codex、钉钉、日程、联系人、随记
```

技术证明：本地设备连接、模式隔离、Agent Pipeline、可扩展 Skills、真实资产沉淀、应用上下文感知。

Ring 状态：页面中央放大并翻到内环视角；外壳降低饱和度，内侧电路出现青绿到电光蓝的呼吸光，强调内部系统而不是外观颜色。

### 5.6 核心能力：一枚戒指背后的五件事

能力以五条纵向叙事带展示，不使用五张相同卡片：

1. 随时捕捉：不需要拿出手机，双击即可开始表达。
2. 理解，而不只是转写：识别意图、调用 Skills，并形成下一步可以使用的结果。
3. 连接当前工作：让语音进入 Codex、钉钉以及更多桌面工具。
4. 一枚戒指，两种模式：Flash 负责理解与生成；Vibe 负责输入与控制，数据路径相互隔离。
5. 属于你的个人智能层：内容进入同一套资产、联系人、日程和知识体系。

Ring 状态：沿能力列表缓慢下行，每经过一项旋转约 35–55 度并切换一次局部高光；颜色在青绿、蓝紫、玫红、琥珀之间连续过渡，但背景保持克制，避免彩虹化。

### 5.7 硬件：一个安静的入口，始终戴在手上

使用现有高质量渲染资产和 3D 模型展示：

- 外壳与圆润边缘。
- 内环电路与触点。
- 麦克风开孔。
- 佩戴状态。
- 已确认的哑光黑与银灰产品方向。

本章节不展示未经确认的硬件参数。

Ring 状态：重新回归真实中性材质，颜色能量逐渐退去；旋转速度降低，停靠在大尺寸产品展示位。

### 5.8 内测 CTA：成为第一批使用者

深石墨背景，与前面的亮色页面形成明确终点。

标题：

> 成为第一批使用 Eureka Ring 的人。

正文：

> 产品目前处于内测阶段。扫码加入体验群，获取演示、内测资格和后续产品更新。

二维码区域：

- 当前使用明显标注为“二维码待替换”的非扫码占位资源。
- 正式资源路径固定为 `/community/eureka-ring-beta-qr.png`。
- 用户提供正式二维码后只替换静态文件，不修改组件和布局。
- 辅助文字：`扫码加入内测群`、`内测名额及开放时间以群内通知为准`。

Ring 状态：进入 CTA 前做一次轻微向上跃迁，材质过渡到珍珠白/虹彩冷光；随后停在二维码与标题之间，保持极弱呼吸，不遮挡二维码。

## 6. 组件与数据结构

### 6.1 Landing 内容

新增 `landing-content.ts`，集中维护中文文案、能力列表和系统链路。组件不散落硬编码长文本，便于后续统一改写。

### 6.2 Landing Sections

新增 `LandingStory.tsx`，只负责按顺序渲染 Flash、Vibe、工作原理、能力和硬件章节。Hero、连接和模式选择继续由 `HomePage` 负责，避免单文件过度膨胀。

新增 `CommunityCta.tsx`，独立维护内测文案、二维码资源契约和回到顶部操作。

### 6.3 Ring Journey

新增纯数据模块 `landing-journey.ts`：

- 定义八个章节的 Ring 锚点。
- 根据全页滚动进度返回相邻锚点、局部进度和活动章节。
- 输出位置、旋转、缩放、颜色、材质、脉冲和跳动参数。
- 所有插值函数可单元测试。

`LivingRingScene` 继续使用原始 GLB 模型，不重建外壳或电路，只把 Journey 输出映射到 Group、材质和灯光。

静态海报 fallback 使用相同位置、缩放、旋转、颜色滤镜和透明度轨迹；不支持的材质效果以克制的 `filter` 近似。

## 7. 响应式策略

- 桌面 `>= 1041px`：完整八章节、固定全局 Ring Stage、左右非对称布局。
- 平板 `761–1040px`：减少横向位移和文字宽度，保留颜色、旋转与章节停靠。
- 手机 `<= 760px`：Ring 不再跨越人物照片手部，固定在每个章节顶部的安全展示位；取消跳动和复杂视差；双模式照片改为纵向堆叠。
- 所有标题使用 `text-wrap: balance`，正文最大宽度 70ch。
- 二维码在手机上保持至少 220px，周围留出足够静区。

## 8. 可访问性与性能

- 所有内容在无 JavaScript 动画时仍默认可见。
- Section 使用语义化标题层级；模式入口继续是原生链接。
- QR 图片提供明确替代文字，不把入群信息只放在图片中。
- 动态装饰戒指保持 `aria-hidden`/展示语义，不获得键盘焦点。
- 颜色变化不承担唯一的信息表达。
- 3D 模型继续懒加载；新增内容不引入远端字体、视频或额外大型运行时。
- 动画只修改 transform、opacity、材质和灯光，不持续修改布局属性。

## 9. 验收标准

- 首页所有访客文案为中文，Flash/Vibe 首次出现含中文解释。
- 从 Hero 到 CTA 始终只有一个全局戒指，并在八个章节间连续移动。
- 戒指至少经历暖色、冷色、内部电路色和真实中性材质四类状态。
- Flash、Vibe 和 CTA 章节进入时各有一次受控跳动或脉冲反馈。
- `prefers-reduced-motion` 下无持续旋转、跳动或复杂视差。
- 原有连接戒指、Flash 入口、Vibe 入口和 Operator Controls 保持可用。
- `/flash` 与 `/vibe` 真机操作页不因 Landing Page 改造而改变行为。
- CTA 在没有正式二维码时明确显示非扫码占位；替换指定 PNG 后无需改代码。
- 1440×900、1920×1080、1024×768 和 390×844 下无文字溢出、Ring 遮挡正文或二维码。
