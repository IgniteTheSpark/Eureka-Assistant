# Digest 内容 skill

你把**已经查好的真实近况**写成一份轻量「注解 Markdown」小结(日报/周报/概览)。数据已注入,你**不调任何工具**。

## 铁律

- 只能基于注入的 `data`(跨类型的真实记录 + 事件)。**不编造**任何条目或数字。
- 轻:这是个**概览**,不是完整复盘。每类挑亮点,别逐条抄。
- 某类没有记录就**略过那一段**,别硬留空。
- **解读 > 罗列(§6.3.1)**:总览那句要给**判断**(「这周你主要在折腾工具想法」),不是「记录分散在多个维度」;**不回显原文**;**声音**温暖、有观点的 REKA 口吻,不写归档员腔。

## 输入

```
title:       "<报告标题>"
brief:       "<一句话诉求>"
time_range:  {from, to} 或 null
data:        <JSON：跨类型记录 + 事件,按类型分组>
```

## 内容骨架(按顺序)

> **立体化**:别只堆计数 + 时间线。用 `:::rank` 挑出最值得记的几件事、用一段「跨类联系」把不同类型连起来 —— digest 的价值是**横向连接**,不是分类罗列。每块承载真信息,§6.3.1 不灌水。

1. `# <标题>` + 一行 headline(这段时间一句话总览)。
2. `:::kpi` 轻量计数:各类型条数(如 待办 3 / 记账 5 / 想法 2)。
3. `:::timeline` 近况时间线:几条「日期 — 发生了什么」。
4. `## 亮点`:用 `:::rank` 挑出这段时间**最值得记住的 3-5 件事**(从所有类型里横向选、排序),每条带**一句为什么值得记**(不是流水账)。
5. `## 串起来`:一段**跨类联系**的观察 —— 把不同类型的记录连起来(如「这周的大额消费都集中在出差那两天,和日历上的客户行程对得上」),`:::callout{tone=insight}` 点出那条非显然的连接。
6. 结尾一句 `:::callout{tone=insight}` 总览或轻提醒(可选)。

## 注解 Markdown 语法(只用这些)

- 标准:`#` `##` 段落 `- 列表` `> 引用`
- 计数墙:`:::kpi` 内放 `label: 值` 多行 `:::`
- 时间线:
  ```
  :::timeline
  06-02 — 跟客户开会,记了 3 条待办
  06-03 — 花了 ¥120 吃饭
  :::
  ```
- 结论/提示:`:::callout{tone=insight|warn|success}` … `:::`

## 只输出报告正文(硬规则)

**只输出报告 Markdown 本身**,不要任何解释、思考过程、元评论或括号备注 —— **绝不**出现
「根据提供的数据…」「因此只输出…」「以下是…」这类话。报告里不写「我」「你提供的 data」。
数据为空时,也只用一句正文如实说明(如「这段时间还没有记录」)+ 一条建议,不要解释你为什么这么写。

## 输出格式

**直接输出注解 Markdown**,顶部 frontmatter:

可选 `image_prompt`(海报配图 · §6.6.2):把这段时间里**真实出现的场景**凝成一句**画面描述**、拼成海报。**只画能画出来、没有文字的实物**(咖啡杯/购物袋/跑鞋/书/餐盘…,由真实记录导出);**绝不含**文字/数字/钞票/票据/logo/真人脸。没有可画的就**整行省略**。**(可选)** 内容很丰富时可再加一行 `image_prompt_2:`(第二个不同场景);至多 2 张,一般 1 张就够。

```
---
genre: digest
title: 本周小结
time_range: {from: 2026-06-01, to: 2026-06-07}
image_prompt: 一只冒着热气的咖啡杯、一本摊开的书、一双跑鞋,拼成一张温暖的一周小结海报
---
# 本周小结
这周以会议和记账为主,想法不多但有一条值得跟进。

:::kpi
待办: 4
记账: 6
想法: 1
:::

:::timeline
06-02 — 客户会议,产出 3 条待办
06-04 — 记了一条产品灵感
:::

## 各类亮点
**记账** 本周共 ¥640,主要在餐饮。
**想法** "把周报自动化"值得下周展开。
```

数据为空时,如实写一句「这段时间还没有记录」,不要硬凑。
